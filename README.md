3. ВАЖНО — КОНФЛИКТ ПОРТОВ POSTGRESQL (ТОЛЬКО ДЛЯ WINDOWS)
================================================================================

На Windows может быть установлен локальный PostgreSQL который занимает порт 5432.
Docker пытается использовать тот же порт — возникает конфликт и node подключается
не к той базе данных.

РЕШЕНИЕ: использовать другой порт для Docker контейнера.

Откройте файл docker-compose.yml и измените строку ports:

  БЫЛО:
    ports:
      - "5432:5432"

  СТАЛО:
    ports:
      - "55433:5432"

Сохраните файл.

4. НАСТРОЙКА ПЕРЕМЕННЫХ ОКРУЖЕНИЯ
================================================================================

Создайте файл .env в папке landing/server/ со следующим содержимым:

  --- МИНИМАЛЬНАЯ КОНФИГУРАЦИЯ (обязательно) ---

  # Если НЕТ конфликта портов (Mac/Linux или чистый Windows без PostgreSQL):
  DATABASE_URL=postgresql://melody:melody@localhost:5432/melody

  # Если ЕСТЬ конфликт портов (Windows с локальным PostgreSQL):
  DATABASE_URL=postgresql://melody:melody@localhost:55433/melody

  # JWT секрет (минимум 16 символов, можно любую строку)
  JWT_SECRET=melody_dev_secret_34563568666754674563434352

  # Порт сервера
  PORT=8787

  --- ДОПОЛНИТЕЛЬНАЯ КОНФИГУРАЦИЯ (для полного функционала) ---

  # CORS (адрес клиента)
  CORS_ORIGIN=http://localhost:5173

  # SMTP для формы обратной связи (Gmail)
  # Получить пароль приложения: Google Аккаунт → Безопасность → Пароли приложений
  SMTP_HOST=smtp.gmail.com
  SMTP_PORT=587
  SMTP_SECURE=false
  SMTP_USER=ваш_gmail@gmail.com
  SMTP_PASS=пароль_приложения_gmail
  SMTP_FROM=ваш_gmail@gmail.com
  FEEDBACK_TO=куда_слать@gmail.com

  # OpenAI API для GPT чата (опционально)
  # Получить ключ: https://platform.openai.com/api-keys
  OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

  4. Нажмите "+ mp3" и загрузите любой mp3 файл

12. ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ
================================================================================

ПРОБЛЕМА: npm error Missing script: "dev" (при запуске сервера)
РЕШЕНИЕ:  Используйте "node index.js" вместо "npm run dev"
          npm run dev работает только для клиента (папка landing/)

ПРОБЛЕМА: [ERROR] DATABASE_URL is not set
РЕШЕНИЕ:  Создайте файл .env в папке landing/server/ (см. раздел 4)

ПРОБЛЕМА: password authentication failed (28P01)
РЕШЕНИЕ:  Конфликт портов PostgreSQL на Windows.
          Измените порт в docker-compose.yml на 55433:5432
          и DATABASE_URL на localhost:55433

ПРОБЛЕМА: listen EADDRINUSE: address already in use :::8787
РЕШЕНИЕ:  Порт занят. Завершите старый процесс:
          Windows: netstat -ano | findstr :8787 → taskkill /PID xxxx /F
          Mac/Linux: lsof -ti:8787 | xargs kill -9

ПРОБЛЕМА: Трек не воспроизводится после загрузки
РЕШЕНИЕ:  Нажмите Play вручную — браузер блокирует автозапуск звука.
          Убедитесь что сервер запущен и база данных работает.

ПРОБЛЕМА: bcrypt ошибка на Mac M1/M2/M3
РЕШЕНИЕ:  cd landing/server && npm install bcrypt --build-from-source

ПРОБЛЕМА: Cannot find module @rollup/rollup-darwin-arm64
РЕШЕНИЕ:  cd landing && rm -rf node_modules package-lock.json && npm install
